What's Wrong with This Picture?

  • Map XML tree to non-XML (incompatible) model
  • Mapping language
    • Not very powerful
    • Syntax leads to misunderstandings
    • Used for every decompose/compose
  • XML hampered by non-XML limitations

Your XML will only ever be as good as your non-XML implementation
